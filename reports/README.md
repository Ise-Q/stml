# Reports — feature-engineering deliverables

This directory holds the human-readable deliverables for the shared
**feature-engineering base**. Modelling and signal-replication write-ups live on
the branches cut from here, not in this directory.

## Contents

1. [`feature-catalog.md`](feature-catalog.md) — **the feature deliverable.**
   Auto-generated by `stml.metamodel.catalog.render_catalog`. One row per
   produced feature column, grouped by family (F1–F17), each with what it
   captures, its lookback, its leakage class (E / TF / LI), and a reuse pointer.
   It opens with the methodology notes: the leakage contract, the two
   standardization regimes (FE-train-frozen TF families vs causal expanding-z
   twins), the F15 daily-sigma note, the provenance of every family
   (signal-deep-dive / Harry / Sreeram), and the no-labels statement.

2. [`missing-data-report.md`](missing-data-report.md) — the authoritative
   data-cleaning reference: every category of missing / anomalous row in the
   futures OHLCV panel and the handling policy `stml.io.load_clean_data` applies
   (drop calendar-impossible weekend rows, keep the 765 valid zero-volume settle
   rows, never forward-fill structural NaNs). Read this once before using the data.

## How the catalog is produced

`reports/feature-catalog.md` is regenerated by the feature build and must stay in
exact 1:1 correspondence with the produced matrix columns (the build asserts this
via `catalog.assert_coverage`):

```bash
uv sync --group features-extra
uv run python -m stml.metamodel.build_features
```

## Leakage classes (recap)

| Class | Meaning |
|---|---|
| **E** | Engineered, no fit; causal by truncation-invariance (value at `t` identical on `data[:t+1]` and `data[:T]`). |
| **TF** | Fitted on the FE-train partition only (ending `2021-07-01`), applied causally with frozen parameters (F3 / F4 / F11 / F16 / F17). |
| **LI** | Label-interface subset (`f2_vol_20` = candidate barrier sigma — **annualised, de-annualise by √252** before using as a triple-barrier sigma; `f5_trailing_run_length`). Engineered + causal, flagged as the FE → label hand-off. |

The matrix itself carries **no label column** — each downstream branch attaches
its own labels.
