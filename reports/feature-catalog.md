# Feature Catalog ‚Äî Triple-Barrier Metamodel

Auto-generated by `stml.metamodel.catalog.render_catalog`. One row per produced feature column; the four provenance columns (`date`, `instrument`, `partition`, `fe_train_end_date`) are metadata, not features.

**71 feature columns** across 9 families (E=54 engineered, TF=15 fitted, LI=2 label-interface).

## Leakage classes

- **E** ‚Äî engineered, no fit; causal by truncation-invariance (value at `t` identical on `data[:t+1]` and `data[:T]`).
- **TF** ‚Äî fitted (GMM / Markov / PCA / KMeans / autoencoder / scaler) on the FE-train partition only (ends 2021-07-01), transformed causally with frozen params.
- **LI** ‚Äî the label-interface subset the deferred triple-barrier label consumes (`f2_vol_20` = barrier sigma; `f5_trailing_run_length` = run membership); engineered + causal, flagged as the FE -> label hand-off.

## Methodology notes

- **F1 is the highest-value family (per C1).** The signal is a short-horizon mean-reversion / counter-trend strategy, so the F1 counter-trend family ‚Äî led by `f1_mr_score_20` ‚Äî is expected to carry the most predictive value.
- **F9 `f9_pair_corr_mean` is expected-negative.** The cross-asset mean absolute correlation is approximately 0.09; the cross-sectional features are retained precisely because they capture near-independent structure the per-instrument time-series features miss.
- **F5 `f5_trailing_run_length` / `f5_days_since_flip` are CAUSAL trailing features** computed on `s[:t+1]` (an expanding scan). They are deliberately DISTINCT from the full-period `splits.run_length_p90`, which measures the entire released window and is reserved for embargo sizing only (it would leak the future if used as a feature).
- **F7 `f7_amihud_20` uses a zero-volume -> NaN guard.** A zero-volume day contributes NaN to the trailing Amihud illiquidity mean rather than dividing by zero; the resulting structural NaNs are never forward-filled.
