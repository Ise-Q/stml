# Results Artifacts

This directory holds machine-readable artifacts generated by the project
pipelines. It has three layers:

- **root-level files** (this README) — the **feature-engineering matrix** and its
  companion maps, produced by the metamodel feature layer
  (`stml.metamodel.build_features`);
- [`jj/`](jj/README.md) — the canonical **signal-replication** JSON artifacts
  (`thresholds.json`, `ledger.json`, `top_candidates.json`);
- `<initials>/` — personal figures / tables / intermediate outputs (not shared
  until a checkpoint).

The reports in `reports/` render these numbers for human reading; the files here
are the structured records. The graded per-column documentation of the feature
matrix lives in [`reports/feature-catalog.md`](../reports/feature-catalog.md).

## How to regenerate

```bash
uv run python -m stml.metamodel.build_features                 # all 11 instruments
uv run python -m stml.metamodel.build_features --instruments si1s   # smoke subset
```

The build is deterministic: rebuilding reproduces the matrix frame-for-frame
(values, dtypes, and column order), with the autoencoder-derived columns equal to
within `1e-10`. Running the CLI does **not** touch `jj/`.

## Files

`feature_matrix.parquet` / `feature_matrix.csv`
: The tidy-long feature matrix. Parquet is canonical; CSV is the convenience
copy. One row per **non-zero-signal trade day** per instrument over the released
window (`2020-01-03` to `2022-06-30`): **4,984 rows** across the 11 instruments,
**75 columns** = 4 metadata + 71 feature columns.

`feature_redundancy.json` / `feature_redundancy.csv`
: The feature redundancy map: a pairwise-complete correlation matrix
(`na_checks.corr_max_info`) over the 71 feature columns, plus a SciPy hierarchical
clustering of `1 - |corr|` and, per feature, its highest-`|corr|` partner. Use it
to see which engineered features are near-duplicates (the deferred
importance phase consumes this map).

`instrument_scope.json`
: The D5 `InstrumentScope` registry — per-instrument fitting-scope decisions and
power statistics. See below.

`feature_matrix_provenance.json`
: The FE→model handoff contract: the FE-train boundary, split boundaries, and a
summary of every fitted artifact's training window. See below.

## feature_matrix layout

Metadata columns (always first):

`date`
: Trade date.

`instrument`
: Ticker (e.g. `es1s`).

`partition`
: `train` / `val` / `test` from the chronological split of the released signal
calendar (train ends `2021-07-01`). This is the **released-data** split for
feature development, not the coursework's hidden Jul–Dec 2022 set.

`fe_train_end_date`
: Constant `2021-07-01` — the boundary every fitted feature was trained on, also
stored in `df.attrs`. Downstream models must respect it (train ≤ this date or
refit through the documented hook).

The 71 feature columns are grouped by family (full per-column documentation in
[`reports/feature-catalog.md`](../reports/feature-catalog.md)):

| Family | Cols | Leakage class | What it captures |
|---|---|---|---|
| F1 counter-trend | 15 | E | distance-from-MA z-scores (the C1 highest-value family), RSI, Bollinger, hi-lo position |
| F2 vol / dispersion | 11 | E (+ `f2_vol_20` is LI) | rolling/Parkinson/Garman-Klass vol, ATR, skew/kurtosis |
| F5 signal-derived | 9 | E (+ `f5_trailing_run_length` is LI) | trailing run-length / days-since-flip on `s[:t+1]`, participation, sign-agreement |
| F6 momentum-contrast | 7 | E | vol-scaled trailing return, MA-cross / MACD, ADX, Donchian position |
| F7 microstructure | 7 | E | volume/OI z-scores & trend, OI-price divergence, Amihud illiquidity |
| F8 calendar | 4 | E | day-of-week and month sin/cos |
| F3 regime | 4 | TF | filtered GMM + Markov-switching high-vol posteriors, switch-prob, dwell |
| F4 latent | 11 | TF | PCA(4), KMeans cluster id/distance, dense-autoencoder code + reconstruction error |
| F9 cross-section | 3 | E | cross-sectional rank, universe size, mean class pair-correlation |

`leakage_class`: **E** engineered (proven causal by truncation-invariance), **TF**
fitted (proven causal by a fit-provenance assertion), **LI** the label-interface
subset earmarked for the deferred triple-barrier label.

## instrument_scope.json

A JSON object keyed by ticker. Each entry records (D5):

`asset_class`
: `EQ` {es1s, nq1s, fesx1s} / `EN` {cl1s, ho1s, rb1s, ng1s} / `ME` {gc1s, si1s, hg1s, pl1s}.

`fit_scope_regime`
: Always `per_instrument` — GMM/Markov regime models fit on the instrument's own
dense daily-return series (~5.6k–7.9k returns), so the signal's thin `n_eff` is
irrelevant to the fit.

`fit_scope_latent`
: Always `pooled_within_class` — PCA/KMeans/autoencoder fit on the asset class's
pooled FE-train engineered rows (EN 668 / EQ 1083 / ME 1174), which representation
learners need to avoid overfitting thin per-instrument blocks.

`n_eff_gate`
: Post-embargo validation effective sample size (number of constant-signal runs),
the reporting/low-power statistic — **not** a fitting-scope trigger.

`low_power`
: `True` when `n_eff_gate < FLOOR` (10): `cl1s`, `ho1s`, `ng1s`.

`embargo_p90`
: The full-period 90th-percentile constant-signal run length — the purge/embargo a
downstream cross-validation should apply at each split boundary.

## feature_matrix_provenance.json

The structured FE→model handoff contract:

```text
fe_train_end_date  : 2021-07-01
split boundaries   : train / val / test date ranges of the released window
regime artifacts   : per-instrument train_index summary (min/max date + count)
latent artifacts   : per-class train_index count
matrix shape       : n rows, n feature columns
seed               : the RNG seed used for the deterministic build
```

It exists so the deferred model phase can verify, mechanically, that every fitted
feature was trained only on data `≤ 2021-07-01`.

## How these files relate to the reports and source

- [`reports/feature-catalog.md`](../reports/feature-catalog.md) documents every
  feature column (family, what it captures, lookback, leakage class, reuse pointer)
  and records the autoencoder-vs-PCA(k=4) reconstruction comparison per class.
- [`src/README.md`](../src/README.md) describes the `stml.metamodel` package that
  produces these artifacts.
- [`notebooks/jay/feature_engineering.ipynb`](../notebooks/jay/feature_engineering.ipynb)
  builds the matrix end-to-end and visualizes the families, regime posteriors,
  AE-vs-PCA comparison, and the redundancy heatmap/dendrogram.
